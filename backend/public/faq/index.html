Frequently Asked Questions

Answers to common questions about nex.

How does nex differ from npm or pip? +

Nex is designed specifically for distributing standalone CLI tools and binaries, rather than library dependencies for projects. It focuses on ease of use for end-users and cross-platform compatibility.

Is nex free to use? +

Yes, nex is open-source software released under the MIT license. You can use it for free for both personal and commercial purposes.

How do I publish my own package? +

To publish a package, you need to create a package definition file and submit a pull request to the nex registry repository. See our documentation for detailed instructions.

Where are packages installed? +

Packages are installed in a dedicated directory in your user home folder (e.g., ~/.nex/packages on Linux/macOS or %USERPROFILE%\.nex\packages on Windows).

Can I use nex offline? +

Nex requires an internet connection to install or update packages. However, once installed, packages can be run offline.
